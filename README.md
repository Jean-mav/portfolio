### 🔹 2. Analyse E-commerce & Segmentation RFM

📌 **Objectif** : Analyser les transactions d'un retailer en ligne (540k+ lignes) pour identifier les leviers de croissance et segmenter la clientèle.

🛠️ **Outils utilisés** : 
- **R / Kaggle** : Nettoyage massif (gestion de 135k valeurs manquantes), filtrage des anomalies et Feature Engineering.
- **Analyse RFM** : Création d'une segmentation par Récence, Fréquence et Montant pour cibler les clients à haute valeur.
- **Tableau Public** : Création d'un dashboard interactif piloté par la saisonnalité et la performance géographique.

📊 **Résultats clés** :
- Identification d'une forte dépendance au marché UK (> 80% du chiffre d'affaires), révélant un risque business et une opportunité de diversification.
- Segmentation RFM directement exploitable pour lancer des campagnes marketing ciblées (fidélisation, réactivation, upsell).

🔗 **Voir le projet** : [Notebook Kaggle](https://www.kaggle.com/code/jninon/online-retail-customer-clustering-analysis)
🔗 **Voir le tableau de bord** : [Tableau Public](https://public.tableau.com/app/profile/jean.mav8294/viz/OnlineRetailPerformanceAnalysis2011/)

### 🔹 3. Dashboard BI "Light" sur Google Sheets

📌 **Objectif** : Répliquer une analyse de données complexe (KPIs et RFM) uniquement avec Google Sheets, pour offrir une solution accessible et interactive sans outils tiers.

🛠️ **Points forts techniques** : 
- **Fonctions QUERY** : Automatisation de l'extraction et de l'agrégation des données.
- **Tableau de bord dynamique** : Création de KPIs (Revenu Total) et graphiques de tendances liés à des segments dynamiques (filtres par pays).
- **Analyse RFM native** : Calcul des scores de segmentation via des formules d'agrégation pures (sans scripts externes).

📊 **Résultats clés** :
- Une solution de reporting 100% gratuite et collaborative.
- Un outil de décision robuste accessible directement dans le Cloud pour les équipes opérationnelles.
- Solution idéale pour les PME ne disposant pas d'outils BI avancés.

🔗 **Voir le projet** : [Google Sheet](https://docs.google.com/spreadsheets/d/1Zzag7MupWSWyCMmUufaHi4K3bIKiWtphrGCtwkvxXLg/edit?usp=sharing)

### 🔹 4. Analyse de Performance Retail (SQL - Chinook)

📌 **Objectif** : Explorer une base de données de vente de musique pour identifier les disparités régionales et évaluer la performance des équipes de vente.

🛠️ **Outils utilisés** : 
- **SQL** : Requêtes complexes (JOINs, Aggrégations, etc.).
- **DB Browser for SQLite** : Gestion et exploration de la base relationnelle.

📊 **Résultats clés** :
- Optimisation Commerciale : Analyse comparative des revenus générés par agent de vente pour identifier les meilleures pratiques internes.
- Insights Marché : Identification de marchés à forte concentration et opportunités de croissance sur les pays à panier moyen élevé.
- Stratégie Contenu : Segmentation des ventes par artistes et morceaux pour orienter les futures campagnes marketing.

🔗 **Voir le projet** : [Github](https://github.com/Jean-mav/chinook-project)
